# Synthetic benchmark population

The production-shaped benchmark population is generated by `scripts/seed-synthetic-population.mjs` and is deliberately separated from normal signup and notification routes.

It creates 100 premium installer identities, 50 premium wholesaler identities and 200 free consumer identities in Firebase Authentication. The generated D1 migration adds clearly marked synthetic profiles, CRM work, products, projects and anonymised opportunity matches without generating real admin alerts, emails, marketing messages or billing events.

Synthetic trade accounts receive active membership and premium feature grants for role-appropriate tools. Synthetic consumers remain free. Consumer emails are intentionally unverified, while the benchmark migration preloads submitted projects so they can be inspected without sending email.

The credentials CSV is generated outside the repository and must never be committed or deployed. Every test email uses the reserved `example.com` domain. Use these identities only for manual QA and performance benchmarks.

To create a new controlled population, run the generator with explicit output and migration paths. Store the credentials securely, review the generated migration, then deploy through the normal Sites migration lifecycle.
